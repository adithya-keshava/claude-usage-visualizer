# Data Models & Entities

## Overview

The application uses Python dataclasses to represent all domain entities. Models define the shape of data flowing through the system from raw files through aggregation to template rendering.

**File Reference:** `src/app/data/models.py`

## Token Usage Model

### TokenUsage

**Location:** `src/app/data/models.py:4-10`

Core token counting dataclass:

```python
@dataclass
class TokenUsage:
    input_tokens: int = 0
    output_tokens: int = 0
    cache_creation_input_tokens: int = 0
    cache_read_input_tokens: int = 0
```

**Purpose:** Represents token usage for a single message or message segment.

**Fields:**
- `input_tokens`: Regular input tokens sent to model
- `output_tokens`: Tokens generated by model
- `cache_creation_input_tokens`: Tokens written to prompt cache
- `cache_read_input_tokens`: Tokens read from prompt cache (free or reduced cost)

**Cost implication:** Each field maps to different pricing:
- input → input_rate
- output → output_rate
- cache_creation → 1.25× input_rate
- cache_read → 0.1× input_rate

See `src/app/data/pricing.py:159-175` for cost calculation.

## Message-Level Models

### SessionMessage

**Location:** `src/app/data/models.py:54-63`

Represents a single assistant message from a session JSONL file:

```python
@dataclass
class SessionMessage:
    timestamp: str              # ISO 8601 format
    model: str                  # Model ID (e.g., "claude-opus-4-6")
    input_tokens: int
    output_tokens: int
    cache_creation_input_tokens: int
    cache_read_input_tokens: int
    cost_usd: float             # Calculated cost at load time
```

**Data source:** Parsed from `projects/{encoded_path}/{session_id}.jsonl` at `src/app/data/loader.py:201-242`

**Field details:**

| Field | Source | Extraction | Calculation |
|-------|--------|-----------|-------------|
| `timestamp` | `message.timestamp` | Direct | ISO 8601 string |
| `model` | `message.message.model` | Nested access | String (e.g., "claude-opus-4-6") |
| `input_tokens` | `message.message.usage.input_tokens` | Nested + parsing | Integer |
| `output_tokens` | `message.message.usage.output_tokens` | Nested + parsing | Integer |
| `cache_creation_input_tokens` | `message.message.usage.cache_creation_input_tokens` | Nested + parsing | Integer |
| `cache_read_input_tokens` | `message.message.usage.cache_read_input_tokens` | Nested + parsing | Integer |
| `cost_usd` | Derived | `estimate_cost()` | Float (2+ decimals) |

**Filtering rules** (see `src/app/data/loader.py:209-217`):

Lines in JSONL are skipped if:
- `type != "assistant"` - Only assistant messages count
- `model == "<synthetic>"` - Synthetic messages don't consume tokens
- `isSidechain == true` - Abandoned branches are excluded

**Cost calculation:** `estimate_cost(model, **parsed_usage)` at `src/app/data/loader.py:226`

Uses pricing from `src/app/data/pricing.py:152-175`:
```python
cost = (
    input_tokens * pricing["input"]
    + output_tokens * pricing["output"]
    + cache_creation_input_tokens * pricing["cache_write"]
    + cache_read_input_tokens * pricing["cache_read"]
) / 1_000_000
```

## Session-Level Models

### SessionSummary

**Location:** `src/app/data/models.py:66-77`

Aggregated data for a single session (user conversation):

```python
@dataclass
class SessionSummary:
    session_id: str
    slug: str                       # User-friendly name
    project: str                    # Original path from history.jsonl
    timestamp: str                  # First message timestamp
    message_count: int              # Number of assistant messages
    total_input_tokens: int         # Sum of all input_tokens
    total_output_tokens: int        # Sum of all output_tokens
    total_cost_usd: float           # Sum of all costs
    models_used: set[str]           # Set of models used in session
```

**Data source:** Built by `build_session_summary()` at `src/app/data/loader.py:256-357`

**Slug determination** (priority order) at `src/app/data/loader.py:281-315`:

1. **Renamed display name:** If user executed `/rename` command in history.jsonl
   - Source: `history[session_id]["display"]` with `renamed=True` flag
   - Allowed length: up to 100 characters
   - Example: User renames "cryptic-id-123" to "Payment Gateway Debugging"

2. **History display name:** First message text from history.jsonl
   - Source: `history[session_id]["display"]` with `renamed=False` flag
   - Cleaned: Remove newlines, truncate to 80 chars
   - Rejected: Slash commands (/rename, /add, etc.), exclamations, "yes", "Sure", "go ahead"
   - Example: "How do I debug async payment processing?"

3. **JSONL slug field:** Field within session JSONL file itself
   - Source: `slug` field from first occurrence in session JSONL
   - Format: Unknown (implementation detail of file format)

4. **Session ID prefix:** Last resort fallback
   - Uses first 8 characters of session_id
   - Example: "a1b2c3d4" from "a1b2c3d4e5f6g7h8"

**Models used:** Aggregated from all messages at `src/app/data/loader.py:272`:
```python
models_used = {m.model for m in messages}
```

**Subagent merging** at `src/app/data/loader.py:317-345`:

If a subagent directory exists at `projects/{encoded_path}/{session_id}/subagents/`:
- Iterates `agent-*.jsonl` files
- Parses assistant messages same as main session
- Adds tokens and costs to session totals
- Adds models to `models_used`
- Maintains same filtering rules (skip synthetic, sidechain)

**Cost calculation:** Sum of message costs from `src/app/data/loader.py:271`:
```python
total_cost = sum(m.cost_usd for m in messages)
```

### ProjectSummary

**Location:** `src/app/data/models.py:80-88`

Aggregated data for a project (directory of sessions):

```python

class ProjectSummary:
    encoded_path: str               # URL-safe encoded directory name
    display_path: str               # For UI display (planned)
    session_count: int              # Number of sessions in project
    total_input_tokens: int         # Sum of all sessions' input
```

**Data source:** Built by `build_project_summaries()` at `src/app/data/loader.py:360-395`

**Encoded path:** Directory name from `projects/` folder
- Example: If file exists at `~/.claude/projects/IdeaProjects%2Fmy-project/`, encoded_path is `IdeaProjects%2Fmy-project`
- Used as URL parameter in routing

**Display path:** Planned for UI display
- Currently unused in code (set to `encoded_path`)
- Intended for human-readable project path

**Aggregation logic** at `src/app/data/loader.py:379-392`:

```python
for encoded_dir in projects_dir.iterdir():
    sessions = []
    for jsonl_file in encoded_dir.glob("*.jsonl"):
        session_id = jsonl_file.stem
        summary = build_session_summary(encoded_dir.name, session_id, history)
        if summary:
            sessions.append(summary)

    if sessions:
        projects[encoded_dir.name] = sessions
```

Returns dict: `{encoded_path: [SessionSummary, ...], ...}`

## Model Hierarchy

```
TokenUsage (primitive)
    ↓
SessionMessage (raw message from file)
    ↓
SessionSummary (aggregated across messages + subagents)
    ↓
ProjectSummary (aggregated across sessions)
    ↓
OverviewStats (aggregated across all projects)
    ├── DailyActivity (daily trends)
    ├── DailyModelTokens (daily model breakdown)
    └── ModelStats (per-model totals)
```

## Caching

All models except SessionMessage are cached in memory with TTL + mtime invalidation.

**Cache mechanism:** `src/app/data/cache.py:1-54`

- SessionMessage: Created fresh for each request but cached per session
- SessionSummary: Built on-demand from session files
- ProjectSummary: Cached with key "project_summaries"
- OverviewStats: Cached with key "stats_cache"

**Cache TTL:** 5 minutes (see `src/app/data/cache.py:5`)

**Invalidation:** If source file's mtime changes, cache entry is discarded

## Type Hints

Models use Python 3.10+ type hints:

- `str`: Text
- `int`: Integer
- `float`: Decimal (USD costs)
- `set[str]`: Unique model IDs
- `dict[str, int]`: Key-value mappings
- `list[T]`: Ordered collections
- `str | None`: Optional fields

## Serialization

Models use `dataclass` decorator without custom serialization.

**For JSON responses:** Convert to dict manually in routers
- Example: `src/app/routers/api.py:44-68` returns dict with "labels" and "datasets"
- Models are intermediate representations, not direct JSON outputs

**For templates:** Pass model instances directly
- Jinja2 templates access fields as `model.field_name`
- Example: `src/app/routers/overview.py:88-96` passes `overview_stats` object
